# Руководство пользователя

Использование веб-интерфейса PSTN Analytics на странице `/ranges`.

Заголовок страницы и вкладки браузера — **«Телефонная нумерация России»**. Имя продукта в репозитории и Docker — **PSTN Analytics**.

---

## Обзор страницы

Главная (и единственная рабочая) страница состоит из:

1. **Панель KPI и действий** — метрики, поиск номера, кнопки
2. **Карточка прогресса импорта** — при загрузке данных
3. **Чипы активных фильтров** — быстрое снятие
4. **Таблица диапазонов** — virtual scroll, фильтры в заголовках

## KPI и панель действий

Пять KPI-карточек показывают **отфильтрованное / глобальное** значение (выборка подсвечена жёлтым):

| Карточка | Формат |
|----------|--------|
| Найденные диапазоны | `N / N` |
| Операторы связи | `N / N` |
| **Регионы / Территории ГАР** | `N (M) / N (M)` — уникальные регионы (уникальные территории ГАР) |
| Суммарная ёмкость | `N / N` |
| **Найти номер** | Маска `(XXX) XXX-XX-XX`; фильтрует диапазоны с пересечением |

### Порядок элементов панели (слева направо)

| Элемент | Описание |
|---------|----------|
| **Сбросить фильтры** | Очищает фильтры, сортировку, **дату датасета** (`asOf`) и **возвращает текущий датасет** (`dataset=current`). Кнопка активна, если есть хотя бы один фильтр, сортировка не по умолчанию, выбран diff snapshot или задана историческая дата |
| **Дата датасета** | Поле **ДД.ММ.ГГГГ** (можно ввести вручную) или календарь по иконке; исторический full-план на выбранную дату |
| **Датасет** | Выпадающий список: **«Датасет DD.MM.YYYY»** (актуальные production-данные) или **«Расхождения DD.MM.YYYY»** (snapshot изменений после import). Список загружается с сервера (`GET /api/datasets`). Подробнее: [import-and-datasets.md](import-and-datasets.md) |
| **БД: X ГБ** | Размер PostgreSQL (`GET /api/storage`) |
| **API** | Справка и готовые curl для external lookup API |
| **XLSX** | Выгрузка текущей выборки в Excel |
| **Загрузить данные** | Полная перезагрузка 4 CSV с opendata.digital.gov.ru |

При пустой базе (до первого импорта) KPI показывают нули и жёлтый баннер с предложением загрузить данные. Если реестр OPR не загружен, показывается отдельное предупреждение об пустой колонке «УВр Антифрод».

## Автообновление данных (production)

В production-окружении (compose с сервисом `pstn_scheduler`) данные **обновляются автоматически** — ежедневно в **12:00 по Москве** запускается cron-импорт четырёх CSV.

| Результат cron | Что видит пользователь |
|----------------|------------------------|
| CSV не изменились | Ничего — import завершился со статусом `skipped`; KPI и таблица без изменений |
| CSV изменились | После import данные обновятся; в селекторе «Датасет» может появиться пункт **«Расхождения DD.MM.YYYY»** |

**Вам не нужно** нажимать «Загрузить данные» каждый день в production. Кнопка нужна для **первой загрузки**, принудительного re-import или в **локальной разработке** (там scheduler отсутствует).

Подробнее: [import-and-datasets.md](import-and-datasets.md).

## Поиск по номеру (маска)

Поле **«Найти номер»** принимает маску в формате `(XXX) XXX-XX-XX`:

- Цифры — фиксированные позиции
- `X` или пустой слот — любая цифра
- Пример: `(383) 399-XX-XX` — все диапазоны ABC 383 с блоком 399xxxx

Фильтрация использует строгое пересечение с диапазоном (функция `phone_mask_match_count` в PostgreSQL), без ложных срабатываний на соседние ABC.

## Таблица диапазонов

### Колонки (режим current / исторический asOf)

| Колонка | Описание |
|---------|----------|
| ABC | Код ABC/DEF (3 цифры) |
| Начало / Конец | Границы диапазона абонентских номеров (7 цифр внутри ABC) |
| Ёмкость | Число номеров в диапазоне |
| Оператор связи | Наименование оператора из CSV |
| Регион | Значение колонки «Регион» из CSV (1:1) |
| Территория ГАР | Значение колонки «Территория ГАР» из CSV (1:1) |
| УВр Антифрод | `id_src` из реестра OPR (JOIN по ИНН); пусто без OPR |
| ИНН | ИНН оператора; ссылка на карточку DaData; подсветка Frontir Network |

### Сортировка

Клик по заголовку колонки — сортировка asc/desc. Поддерживаемые колонки: ABC, Начало, Конец, Ёмкость, Оператор, Регион, Территория ГАР, ИНН.

### Прокрутка

Таблица подгружает данные порциями (infinite scroll). При большом числе строк прокрутите вниз для подгрузки следующей страницы.

### ABC gaps (красные разделители)

Между строками одного ABC кода красная линия означает **пропуск** в нумерации относительно соседнего диапазона (данные вычисляются при import). Маркеры показываются **только в текущем датасете**; в режиме «Расхождения» они отключены.

## Фильтры

### Текстовые фильтры (подстрока / точное число)

| Поле | Поведение |
|------|-----------|
| Начало, Конец | Числовой prefix или подстрока |
| Ёмкость | Точное число или подстрока |

### Фасетные фильтры (combobox с поиском)

- ABC, Оператор, Регион, Территория ГАР, ИНН, УВр Антифрод
- В режиме расхождений дополнительно: **Изменения** (`filters.changedFields`)
- Показывают count по текущей выборке
- Поиск внутри списка значений (debounce 300 ms)

### Coverage AND (ABC, Оператор, Регион, Территория ГАР)

При **мультивыборе** в этих колонках применяется семантика **«оператор должен иметь все выбранные значения»**:

- Пример: ABC `301` + `353` → только операторы, у которых есть **оба** ABC-кода; в таблице видны строки обоих кодов таких операторов
- Одиночный выбор — обычный фильтр «значение равно»

Лимит: до **20** значений на coverage-колонку.

### OR-фильтры (ИНН, УВр Антифрод, Изменения)

Мультивыбор = **любое** из выбранных значений (логическое OR).

Лимит: до **50** значений.

### URL и история браузера

Фильтры, сортировка, выбранный датасет и дата (`asOf`) сохраняются в URL (`filters.*`, `sort`, `dataset=current|diff:<uuid>`, `asOf=YYYY-MM-DD`). Кнопка «Назад»/«Вперёд» браузера восстанавливает состояние.

## Календарь и исторический датасет

Поле **«Дата датасета»** (формат **ДД.ММ.ГГГГ**) и календарь по иконке — **полноценный выбор даты**, не заглушка:

- Дату можно **написать в поле** (Enter или уход фокуса) или **выбрать в календаре** (три месяца в popover).
- **Синий фон** у дня в календаре — день версии датасета (первая загрузка или import с сохранённым snapshot). Даты списка загружаются с `GET /api/datasets/change-dates`.
- Даты **раньше первой загрузки** и **будущие** недоступны для выбора.
- Выбор даты показывает **полный план нумерации**, актуальный на эту дату: берётся последняя сохранённая версия с `load_date ≤ выбранной даты`.
- Таблица, фильтры, KPI и XLSX — как у текущего датасета (9 колонок).
- Под KPI — баннер «План на DD.MM.YYYY (версия от …)»; ссылка возвращает к текущему production.

**Текущий датасет** в селекторе — всегда последний успешный import. Календарь не заменяет селектор, а дополняет его для просмотра прошлых версий.

При выборе исторической даты селектор «Датасет» автоматически переключается на **current** (diff snapshot и asOf не сочетаются).

## Просмотр расхождений (diff snapshot)

В выпадающем списке **«Датасет»** доступны пункты вида **«Расхождения DD.MM.YYYY»** — строки, которые изменились при конкретном import с расхождениями. Пункт появляется **только если import нашёл изменения** в диапазонах; если CSV изменились, но diff пустой, snapshot не создаётся.

**«Датасет DD.MM.YYYY»** — это актуальная production-таблица; дата в подписи соответствует дню последнего **успешного** import (MSK).

| Цвет строки | Тип (`changeType`) |
|-------------|-------------------|
| Зелёный | Добавленный диапазон (`added`) |
| Жёлтый | Изменённый — отличаются метаданные на сегменте (`changed`) |
| Красный | Удалённый фрагмент (`removed`) |

Жёлтая строка (`changed`) появляется **только если на атомарном сегменте отличаются метаданные**: оператор, ИНН, регион, территория ГАР или ёмкость при том же span. Если единственное отличие — «обрезка» родительских границ CSV при **идентичных** метаданных, такой сегмент **не показывается**.

**Колонки в diff-режиме (UI):** ABC, Начало, Конец, Ёмкость, Оператор связи, Регион, Территория ГАР, УВр Антифрод, ИНН, **Статус**, **Изменения**.

| Колонка | Содержимое |
|---------|------------|
| **Статус** | Тип события: «Новый ресурс», «Изменение ресурса» или «Удаление ресурса» |
| **Изменения** | Для строк `changed` — какие поля отличаются (например «Оператор связи, ИНН»); для `added` / `removed` — «—» |

По клику на **Статус** (любая строка) или **Изменения** (только `changed`) открывается окно детализации **«Предыдущая версия / Новая версия»** с пояснением типа события. Отдельных колонок «Старый/Новый оператор» и «Старый/Новый ИНН» в таблице **нет** — они есть только в **XLSX**.

Фильтры в шапках **Статус** и **Изменения** работают как facet-фильтры (OR, до 50 значений). В выпадающем списке показываются **только значения, присутствующие в текущей выборке** (с учётом остальных активных фильтров). **Статус:** `added`, `changed`, `removed`. **Изменения:** оператор, регион, территория ГАР, ИНН.

Экспорт XLS в diff-режиме по-прежнему содержит колонки «Тип изменения», «Старый/Новый оператор связи», «Старый/Новый ИНН» и остальные поля для детального анализа в Excel.

В режиме расхождений работают те же фильтры, KPI, facets (включая Coverage AND), экспорт XLSX. Легенда цветов — **под таблицей**.

Если в URL указан несуществующий snapshot, UI автоматически вернётся к текущему датасету и покажет уведомление.

## Импорт данных

### Запуск

Кнопка **«Загрузить данные»** → `POST /api/import` → фоновый job.

### Источники

Четыре файла с opendata.digital.gov.ru:

| Файл | Примерный объём |
|------|-----------------|
| ABC-3xx.csv | ~69k строк |
| ABC-4xx.csv | ~270k строк |
| ABC-8xx.csv | ~70k строк |
| DEF-9xx.csv | ~15k строк |

**Итого ~446k диапазонов.** Время: ~5–10 минут в зависимости от канала и CPU.

### Этапы (карточка прогресса)

1. Сравнение SHA256 источников с текущим датасетом — при совпадении карточка **«Данные актуальны»** (skip)
2. Подготовка staging
3. Загрузка каждого CSV (4 файла)
4. Проверка полноты (100% строк, минимальные пороги)
5. Расчёт пропусков ABC
6. Анализ расхождений по диапазонам
7. Атомарная подмена production + пересборка справочников
8. Сохранение snapshot расхождений (только если найдены изменения)
9. Привязка УВр Антифрод по ИНН

| Статус карточки | Значение |
|-----------------|----------|
| Синяя «Данные актуальны» | CSV не изменились (`skipped`) — без сетки файлов |
| Зелёная «Загрузка завершена» | Production обновлён |
| Красная | Ошибка; production не изменён (если ошибка до подмены) |

### Политика целостности

- Парсер **не** пропускает строки — любой skip → import failed
- Staging валидируется до swap: parsed count = staging count для каждого файла
- Production **не** обновляется при неполном import
- Swap + rebuild dictionaries — **одна транзакция**

### Прерывание и recovery

Jobs без heartbeat дольше **45 минут** помечаются failed при следующем старте import или перезапуске сервера. При сомнениях — повторите «Загрузить данные».

## Экспорт XLSX

Кнопка **«XLSX»** выгружает текущие **отфильтрованные** данные.

| Порог | Поведение |
|-------|-----------|
| > 100 000 строк | Предупреждение с подтверждением |
| > 500 000 строк | Отказ API `EXPORT_TOO_LARGE` |

Файл: `ranges-export.xlsx`. В **current** — колонки как в таблице. В **diff** — «Тип изменения», old/new оператор и ИНН, остальные поля как в diff-таблице. Ячейки санитизируются от formula injection.

## External API (диалог «API»)

Кнопка **API** открывает справку по read-only lookup API:

- **Точный номер** — `GET /api/v1/lookup?phone=<10 цифр>`, один диапазон или 404. Параметр `dataset` **не поддерживается** — только current production.
- **Поиск по маске** — `GET /api/v1/lookup/search?phone=<маска>`, как «Найти номер», с пагинацией. Поддерживает **`?dataset=diff:<uuid>`** для просмотра расхождений через API.

Готовые curl генерируются сервером (`/api/v1/lookup/examples`) с **реальным ключом системы** — команды можно копировать и выполнять без ручной подстановки.

Авторизация lookup: `Authorization: Bearer <ключ>` или `X-Api-Key`.

Подробнее: [api-reference.md](api-reference.md), [import-and-datasets.md](import-and-datasets.md).

## Сообщения об ошибках и ограничениях

Интерфейс показывает **русскоязычные** сообщения (toast, баннеры, текст в таблице). Технические коды API (`RATE_LIMITED`, `EXPORT_TOO_LARGE` и т.д.) пользователю не выводятся напрямую.

### Rate limit (429)

При слишком частых запросах к import, export, facets или lookup API:

> **Слишком много запросов. Повторите через N сек.**

`N` берётся из заголовка `Retry-After`. Лимиты in-app (с v0.3.19): import — 3 req / 10 min; export — 10 req / min; facets — 60 req / min; lookup — 120 req / min. Подробнее: [security.md](security.md#in-app-rate-limiting).

### Экспорт — слишком много строк

При попытке выгрузить более **500 000** строк:

> **Слишком много строк для экспорта. Сузьте фильтры.**

(В ответе API также указываются точные числа лимита и найденных строк.)

### Ошибка импорта

Красная карточка прогресса с текстом на русском. Типичные формулировки:

| Ситуация | Сообщение |
|----------|-----------|
| Общий сбой | «Загрузка не завершена. Текущие данные в таблице не изменены. Попробуйте снова или обратитесь к администратору.» |
| Прерывание / timeout | «Загрузка прервана из‑за перезапуска сервера или превышения времени ожидания. Текущие данные в таблице не изменены.» |
| Неполные CSV | «Не все обязательные файлы Минцифры загружены или данные неполные. Попробуйте позже.» |
| Пропущенные строки при парсинге | «При загрузке CSV пропущены строки данных. Попробуйте запустить загрузку снова.» |
| Ошибка скачивания | «Не удалось скачать файл с opendata.digital.gov.ru. Проверьте доступность источника и повторите попытку.» |
| Не удалось запустить job | «Не удалось запустить загрузку данных.» |

Production **не изменяется**, если ошибка произошла до атомарной подмены таблицы.

### Пустая база (до первого import)

Жёлтый баннер под KPI:

> **Данные ещё не загружены.** Нажмите «Загрузить данные» для полной загрузки всех четырёх CSV с opendata.digital.gov.ru.

KPI показывают нули.

### Реестр OPR не загружен

Жёлтый баннер:

> **Колонка «УВр Антифрод» пустая:** реестр операторов OPR ещё не загружен в базу. Обычно он подтягивается автоматически при старте приложения.

### Снимок расхождений не найден

Если в URL указан удалённый или неверный `dataset=diff:<uuid>`:

> **Снимок расхождений не найден, показан текущий датасет**

(Toast; таблица переключается на current.)

### Неверная дата датасета

Под полем даты красная подпись:

> **Неверная дата или дата раньше первой загрузки**

При ошибке загрузки списка дат в календаре:

> **Не удалось загрузить даты версий. Обновите страницу или повторите позже.**

### Ошибки загрузки таблицы и фильтров

| Место | Сообщение |
|-------|-----------|
| Первая загрузка таблицы | «Не удалось загрузить данные» |
| Подгрузка следующей страницы (scroll) | «Не удалось загрузить следующую страницу» |
| Facet-фильтры | «Не удалось загрузить фильтры» |
| KPI | «Ошибка загрузки KPI» |
| Некорректные фильтры / cursor (API) | «Некорректные параметры фильтра или сортировки.» / «Некорректный курсор постраничной навигации.» |
| Экспорт | «Ошибка экспорта XLSX» (если нет более конкретного текста от API) |

### Прочие HTTP-ошибки

| HTTP | Типичное сообщение в UI |
|------|-------------------------|
| 401 | «Доступ запрещён.» |
| 404 (не snapshot) | «Запрошенные данные не найдены.» |
| 500 | «Внутренняя ошибка сервера. Попробуйте позже.» |
| 503 | «Сервис временно недоступен. Попробуйте позже.» |

External lookup API (`/api/v1/lookup*`) по-прежнему возвращает **англоязычные** сообщения в JSON — он предназначен для программных клиентов.

## Территории из CSV (1:1)

При импорте колонки **«Регион»** и **«Территория ГАР»** из CSV Минцифры записываются в таблицу **без преобразований** (только trim пробелов). Фильтры «Регион» и «Территория ГАР» работают одинаково, с взаимным сужением (Coverage AND).

После обновления с этой схемой выполните **полный re-import** («Загрузить данные»).

## Реестр OPR (УВр Антифрод)

Колонка **УВр Антифрод** заполняется из таблицы `operators_register` (JOIN по ИНН). Файл реестра **встроен в Docker-образ** (`data/opr/OPR_2026_06_18_00_00_00.csv`) и загружается автоматически при старте и после import CSV Минцифры.

Ручной import (если нужен другой файл):

```bash
tsx scripts/import-opr-csv.ts /path/to/opr.csv
```
