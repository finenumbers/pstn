# Руководство пользователя

Использование веб-интерфейса PSTN Analytics на странице `/ranges`.

---

## Обзор страницы

Главная (и единственная рабочая) страница — **«Телефонный план нумерации России»**. Состоит из:

1. **Панель KPI и действий** — метрики, поиск номера, кнопки
2. **Карточка прогресса импорта** — при загрузке данных
3. **Чипы активных фильтров** — быстрое снятие
4. **Таблица диапазонов** — virtual scroll, фильтры в заголовках

---

## KPI и панель действий

| Элемент | Описание |
|---------|----------|
| Дата загрузки | Время последнего успешного импорта |
| KPI (глобальные) | Число диапазонов, суммарная ёмкость, уникальные регионы и операторы по всей базе |
| KPI (отфильтрованные) | Те же метрики с учётом активных фильтров |
| **Найти номер** | Маска `(XXX) XXX-XX-XX`; фильтрует диапазоны с пересечением |
| **Сбросить фильтры** | Очищает все фильтры и сортировку |
| **API** | Справка и готовые curl для external lookup API |
| **Экспорт XLSX** | Выгрузка текущей выборки |
| **Загрузить данные** | Полная перезагрузка 4 CSV с opendata.digital.gov.ru |

При пустой базе (до первого импорта) KPI показывают нули.

---

## Поиск по номеру (маска)

Поле **«Найти номер»** принимает маску в формате `(XXX) XXX-XX-XX`:

- Цифры — фиксированные позиции
- `X` или пустой слот — любая цифра
- Пример: `(383) 399-XX-XX` — все диапазоны ABC 383 с блоком 399xxxx

Фильтрация использует строгое пересечение с диапазоном (функция `phone_mask_match_count` в PostgreSQL), без ложных срабатываний на соседние ABC.

## Таблица диапазонов

### Колонки

| Колонка | Описание |
|---------|----------|
| ABC | Код ABC/DEF (3 цифры) |
| Начало / Конец | Границы диапазона абонентских номеров (7 цифр внутри ABC) |
| Ёмкость | Число номеров в диапазоне |
| Оператор связи | Наименование оператора из CSV |
| Регион | Субъект РФ (после парсинга территории) |
| Населённый пункт | Населённый пункт (после парсинга) |
| УВр Антифрод | `id_src` из реестра OPR (JOIN по ИНН); пусто без OPR |
| ИНН | ИНН оператора; ссылка на карточку DaData; подсветка Frontir Network |

### Сортировка

Клик по заголовку колонки — сортировка asc/desc. Поддерживаемые колонки: ABC, Начало, Конец, Ёмкость, Оператор, Населённый пункт, Регион, ИНН.

### Прокрутка

Таблица подгружает данные порциями (infinite scroll). При большом числе строк прокрутите вниз для подгрузки следующей страницы.

### ABC gaps (красные разделители)

Между строками одного ABC кода красная линия означает **пропуск** в нумерации относительно соседнего диапазона (данные вычисляются при импорте).

## Фильтры

### Текстовые фильтры (подстрока / точное число)

| Поле | Поведение |
|------|-----------|
| Начало, Конец | Числовой prefix или подстрока |
| Ёмкость | Точное число или подстрока |

### Фасетные фильтры (combobox с поиском)

- ABC, Оператор, Регион, Населённый пункт, ИНН, УВр Антифрод
- Показывают count по текущей выборке
- Поиск внутри списка значений (debounce 300 ms)

### Coverage AND (ABC, Оператор, Регион, Населённый пункт)

При **мультивыборе** в этих колонках применяется семантика **«оператор должен иметь все выбранные значения»**:

- Пример: ABC `301` + `353` → только операторы, у которых есть **оба** ABC-кода; в таблице видны строки обоих кодов таких операторов
- Одиночный выбор — обычный фильтр «значение равно»

Лимит: до **20** значений на coverage-колонку.

### OR-фильтры (ИНН, УВр Антифрод)

Мультивыбор = **любое** из выбранных значений (логическое OR).

Лимит: до **50** значений.

### Пустой населённый пункт

В фасете «Населённый пункт» можно выбрать пустое значение («Не указан») для строк без settlement.

### URL и история браузера

Фильтры и сортировка сохраняются в URL (`filters.*`, `sort`). Кнопка «Назад»/«Вперёд» браузера восстанавливает состояние. Ссылку можно копировать и отправлять коллегам.

## Импорт данных

### Запуск

Кнопка **«Загрузить данные»** → `POST /api/import` → фоновый job.

### Источники

Четыре файла с opendata.digital.gov.ru:

| Файл | Примерный объём |
|------|-----------------|
| ABC-3xx.csv | ~69k строк |
| ABC-4xx.csv | ~270k строк |
| ABC-8xx.csv | ~70k строк |
| DEF-9xx.csv | ~15k строк |

**Итого ~446k диапазонов.** Время: ~5–10 минут в зависимости от канала и CPU.

### Этапы (карточка прогресса)

1. Подготовка staging
2. Загрузка каждого CSV (4 файла)
3. Проверка полноты (100% строк, минимальные пороги)
4. Расчёт пропусков ABC
5. Атомарная подмена production + пересборка справочников
6. Привязка УВр Антифraud по ИНН

### Политика целостности

- Парсер **не** пропускает строки — любой skip → import failed
- Staging валидируется до swap: parsed count = staging count для каждого файла
- Production **не** обновляется при неполном import
- Swap + rebuild dictionaries — **одна транзакция**

### Прерывание и recovery

Jobs без прогресса дольше **10 минут** помечаются failed при следующем старте import или перезапуске сервера. При сомнениях — повторите «Загрузить данные».

## Экспорт XLSX

Кнопка **«Экспорт XLSX»** выгружает текущие **отфильтрованные** данные.

| Порог | Поведение |
|-------|-----------|
| > 100 000 строк | Предупреждение с подтверждением |
| > 500 000 строк | Отказ API `EXPORT_TOO_LARGE` |

Файл: `ranges-export.xlsx`, колонки как в таблице. Ячейки санитизируются от formula injection.

## External API (диалог «API»)

Кнопка **API** открывает справку по read-only lookup API:

- **Точный номер** — 10 цифр, один диапазон или 404
- **Поиск по маске** — как «Найти номер», с пагинацией

Готовые curl генерируются сервером (`/api/v1/lookup/examples`); маска из поля «Найти номер» подставляется в пример search.

Авторизация lookup: `Authorization: Bearer <ключ>` или `X-Api-Key`.

Подробнее: [api-reference.md](api-reference.md).

## Парсинг территорий из CSV

При импорте из CSV Минцифры используются колонки **«Регион»** и **«Территория ГАР»**. Логика [`packages/import/csvParser.ts`](../packages/import/csvParser.ts):

### Территория ГАР

- Спецслучаи (Москва, СПб, Севастополь, Байконур, РФ) — фиксированные settlement/region
- Без `|` — всё в region, settlement пустой
- С `|` — до первого `|` → settlement, после последнего `|` → region

### Fallback на колонку «Регион»

Если region из ГАР содержит `м.р-н`, берётся split колонки «Регион» (`до |` → settlement, `после |` → region).

### Нормализация

- Префиксы `г.`, `г.о.` и т.д. в settlement
- `обл.` → `область` в region
- Перестановка «Междуреченский м.р-н» → «м.р-н Междуреченский»

## Реестр OPR (УВр Антифraud)

Колонка **УВр Антифрод** заполняется из таблицы `operators_register` (JOIN по ИНН). Эта таблица **не** входит в UI-import CSV Минцифры.

Загрузка OPR — отдельная операция администратора:

```bash
tsx scripts/import-opr-csv.ts /path/to/opr.csv
```

После import CSV Минцифры выполняется автоматическая привязка `refreshUvrAntifraudBinding`. Без OPR-файла колонка будет пустой.
